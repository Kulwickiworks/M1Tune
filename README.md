## Getting Started with GitHub

1. Download [GitHub Desktop](https://desktop.github.com/download/)
2. Go to the repository on the GitHub website
3. Click the green **Code** button
4. Press **"Open in GitHub Desktop"**
5. Clone the repository to a folder on your computer
6. Open the **dobby** branch — this is the only branch you should be making changes to

**Your exact steps every time you make changes:**

> File → Open Package → Export As → Name your file using the format below → Put into your GitHub folder in dobby to the moon → Add description and summary → Commit and Push → Update your README

**File naming format:**

> `Initials_V#_M1Tune`

**Examples:**

> `AB_V1_M1Tune`  
> `AB_V2_M1Tune`  
> `AB_V3_M1Tune`

Every time you make a new set of changes, increment the version number. Do not overwrite your previous file.

---

## Pushing Changes

- Keep your files in your **respective folder**
- **Always write a description of what you changed before you push.** Do not push without explaining what you did — nobody should have to go searching through the files to figure out what changed.

## M1 Tune Workflow Instructions

When you finish your section of the M1 tune file, go to **File → Export As** and name your file using the format `Initials_V#_M1Tune` (e.g. `AB_V1_M1Tune`).

Once you have pushed, go to your personal folder's README and log everything you changed using the format below.

> [!IMPORTANT]
> Put **two spaces** at the end of each line to create a new line in GitHub Markdown. Without them, your text will run together.

---

**Variable Name:** Original Value -> New Value  
What this variable does and why you changed it. If the variable needs more in depth research, link it using the format in the [Research](#research) section below.

---

**Example:**

**Main Servo Offset:** 12 -> 15  
Controls the neutral position of the main servo. Increased to correct steering drift during straight-line testing.

---

This is tedious but it ensures you understand what you are doing and allows anyone to read what the variables do without needing to do extra research.
